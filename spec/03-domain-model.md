# 03 — Domain Model & Database Schema (extracted from Flowplane v1)

Sources: `/tmp/flowplane-v1/migrations/` (100 migration files, applied in order to produce the
effective schema below), `src/domain/`, `src/storage/repositories/`, `src/validation/`.
Database is PostgreSQL (TIMESTAMPTZ, BYTEA, JSONB, BIGSERIAL, TEXT[], plpgsql triggers, pgcrypto).

---

## 1. ID strategy, timestamps, JSON conventions

### 1.1 IDs (`src/domain/id.rs`)

All entity IDs are **UUID v4 rendered as TEXT** (`Uuid::new_v4().to_string()`), wrapped in
newtypes generated by a `domain_id!` macro:

```rust
#[serde(transparent)]
pub struct ClusterId(String);   // .new() -> uuid v4 string
```

- 19 newtypes: `ClusterId, RouteConfigId, ListenerId, TokenId, UserId, TeamId, FilterId,
  ScopeId, VirtualHostId, RouteId, EndpointId, ProxyCertificateId, SecretId,
  CustomWasmFilterId, McpToolId, RouteMetadataId, DataplaneId, OrgId, InvitationId`.
- Constructors: `new()` (uuid v4), `from_string`/`from_str_unchecked` (**no validation**),
  `parse()` (validates UUID format only). `FromStr` goes through `parse`.
- Implements `Display`, `AsRef<str>`, `From<String>`, serde-transparent, and sqlx
  `Type/Encode/Decode<Postgres>` as TEXT. `Default` generates a **new random ID** (footgun).
- No prefixes (`clu_…` etc.) — bare UUID strings. A handful of seeded rows historically used
  human-readable IDs (e.g. `'scope-tokens-read'` in the now-dropped `scopes` table) and
  migration-generated IDs use `md5(...)`/`'migrated-' || md5(...)` (grants backfill), so the
  ID columns must accept arbitrary TEXT, not strictly UUIDs.
- Counter-pattern exceptions: `audit_log.id`, `inferred_schemas.id`,
  `aggregated_api_schemas.id`, `rate_limit_domains.id`, `rate_limit_policies.id` are
  `BIGSERIAL` integers; `instance_apps` is keyed by app name (`app_id TEXT`);
  `rate_limit_team_overrides` has a composite PK `(org, team)`.

### 1.2 Timestamps

- Convention: `created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP` (or `NOW()`),
  `updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP` maintained by application code
  (no DB trigger), domain side `chrono::DateTime<Utc>`.
- **Exception (smell):** `proxy_certificates` stores `issued_at/expires_at/revoked_at/created_at`
  as **TEXT** (ISO-8601 strings), and `dataplanes.certificate_expires_at` is TEXT
  (`migrations/20251208000001`, `20260204000001`).

### 1.3 JSON columns

JSON is stored in plain `TEXT` columns (serde_json-serialized) except
`rate_limit_policies.descriptors` which is real `JSONB`. Shapes (from `src/domain/*`):

| Column | Shape |
|---|---|
| `clusters.configuration` | `ClusterSpec` JSON: `{name, endpoints:[{address:{Ip\|Hostname}, port, health_status}], load_balancing (RoundRobin\|LeastRequest{choice_count}\|Random\|RingHash{minimum_ring_size}\|Maglev{table_size}), connect_timeout_seconds, tls_config?, health_checks:[…], circuit_breaker?, outlier_detection?}` |
| `route_configs.configuration` | Envoy-shaped RouteConfiguration JSON: `{name, virtualHosts:[{name, domains:[…], routes:[{name?, match{path strategy, headers, query}, action{cluster\|weightedClusters, pathRewrite?, timeout?, retryPolicy?}}]}]}` (domain types in `src/domain/route.rs`; the normalized `virtual_hosts`/`routes` tables are synchronized projections of this JSON) |
| `listeners.configuration` | `ListenerSpec`-shaped JSON: bind address/port, protocol (`Http\|Http2\|Https\|Tcp`), filter chains incl. inline route config or RDS reference, `tls_config?`, isolation mode |
| `filters.configuration` | Tagged union `{"type": "<filter_type>", "config": {…}}` — custom `Serialize/Deserialize` on `FilterConfig` (`src/domain/filter.rs`); unknown types fall through to `Custom{type, config}` |
| `route_config_filters.settings`, `virtual_host_filters.settings`, `route_filters.settings` | `{"behavior": "use_base"\|"disable"\|"override", "config": {…}, "requirementName": "…"}` (JWT only) — `migrations/20251216000001` |
| `clusters.tags`, `listeners.tags` | JSON array of strings |
| `route_configs.headers` | JSON header-match list for header-based routing |
| `teams.settings`, `organizations.settings` | free-form JSON object |
| `learning_sessions.http_methods` | JSON array, e.g. `["GET","POST"]`; NULL = all |
| `learning_sessions.configuration_snapshot` | JSON snapshot of config at session start |
| `inferred_schemas.request_schema/response_schema`, `aggregated_api_schemas.request_schema` | JSON Schema Draft 2020-12 |
| `aggregated_api_schemas.response_schemas` | object keyed by status code: `{"200": {…}, "404": {…}}` |
| `aggregated_api_schemas.breaking_changes` | JSON array of breaking-change objects |
| `*.request_headers`, `*.response_headers` (schema tables) | `[{"name":"Authorization","example":"Bearer ***"}, …]`, sensitive values redacted at capture (`migrations/20260225000002`) |
| `mcp_tools.input_schema/output_schema`, `route_metadata.request_body_schema/response_schemas` | JSON Schema / status-code map |
| `custom_wasm_filters.config_schema/per_route_config_schema/ui_hints` | JSON Schema objects; `attachment_points` JSON array, default `'["listener", "route"]'` |
| `xds_nack_events.resource_names` | JSON array of rejected resource names |
| `instance_apps.config` | per-app JSON config |
| `secrets.configuration_encrypted` | **BYTEA**: AES-256-GCM ciphertext of the `SecretSpec` JSON (tag `"type"`); `nonce` BYTEA holds the GCM nonce; `encryption_key_id` selects the key |
| `rate_limit_policies.descriptors` | JSONB **flat string map** `{"key":"value",…}` enforced by CHECK `jsonb_is_flat_string_map` |

## 2. Entity catalog

Domain layer (`src/domain/`) is intentionally small; most persistence types are
`*Data`/`*Row` structs in `src/storage/repositories/*.rs` mirroring table columns 1:1.

### 2.1 Tenancy & identity

| Entity | Table | Meaning / lifecycle |
|---|---|---|
| Organization | `organizations` | Outer tenant boundary. Status `active\|suspended\|archived` (CHECK). Special org named `platform` (renamed from `default`, `migrations/20260212000001`) is governance-only, not a tenant. Hard delete (RESTRICTed by teams). |
| OrganizationMembership | `organization_memberships` | user↔org with role `owner\|admin\|member\|viewer` (CHECK). One row per (user, org). |
| User | `users` | Humans and machines. `user_type` `human\|machine` (CHECK, `migrations/20260306000001` Part D). `is_admin` = platform superadmin. `zitadel_sub` UNIQUE links to Zitadel JWT `sub` (JIT provisioning; PAT tables were dropped in `migrations/20260228100001`). `agent_context` (`cp-tool\|gateway-tool\|api-consumer`, CHECK, NULL for humans) classifies machine users. `password_hash` still NOT NULL (legacy of pre-Zitadel auth). Status TEXT default `active` (no CHECK). |
| Team | `teams` | Primary isolation unit. Name unique **per org** (`UNIQUE(org_id,name)` since `migrations/20260208000001`). `envoy_admin_port` BIGINT, globally UNIQUE — auto-allocated per-team Envoy admin port. Status `active\|inactive\|archived` (doc comment only, **no CHECK**). |
| UserTeamMembership | `user_team_memberships` | user↔team link. The `scopes` JSON column was migrated into `grants` and dropped (`migrations/20260306000002`); the row now only asserts membership. |
| Grant | `grants` | Unified authorization grant (replaced `scopes`, `agent_grants`, membership scopes — `migrations/20260306000001`). `grant_type` `resource\|gateway-tool\|route`; resource grants carry `(resource_type, action)`, gateway/route grants carry `route_id` + optional `allowed_methods TEXT[]` (NULL = all methods). Optional `expires_at`. |

### 2.2 Gateway resources (xDS)

| Entity | Table | Meaning / lifecycle |
|---|---|---|
| Cluster | `clusters` | Envoy upstream cluster. `configuration` JSON is source of truth; `cluster_endpoints` is a normalized projection. `source` `native_api\|openapi_import` (CHECK). `version` bumped on every update (drives xDS). |
| ClusterEndpoint | `cluster_endpoints` | Normalized endpoint (address/port/weight/priority/health) extracted from cluster JSON. `health_status` CHECK `healthy\|unhealthy\|degraded\|unknown`. |
| RouteConfig | `route_configs` (renamed from `routes` in `migrations/20251205100000`) | Envoy RouteConfiguration. Owns `virtual_hosts` → `routes` hierarchy, synchronized from `configuration` JSON on create/update. `cluster_name` FK → `clusters(name)` ON DELETE CASCADE (legacy single-cluster column). `route_order`, `headers`, `import_id` support OpenAPI import. |
| VirtualHost | `virtual_hosts` | Per-RouteConfig vhost; `domains` JSON array; `rule_order` = position. |
| Route | `routes` (renamed from `route_rules`) | Individual route rule in a vhost. `match_type` CHECK `prefix\|exact\|regex\|path_template\|connect_matcher`. Names required (auto-generated from path if absent, `RouteMatchType::generate_rule_name`: `prefix-<sanitized>`, `exact-<sanitized>`, `regex-<8-hex-sha256>`, `template-<sanitized>`, `connect`). `exposure` `internal\|external` (CHECK) gates agent grants. |
| Listener | `listeners` | Envoy listener. `address` + nullable `port` (NULL for unix sockets), partial unique on `(address,port)`. `protocol` TEXT default `'HTTP'` (no CHECK). `dataplane_id` FK→dataplanes SET NULL. |
| ListenerRouteConfig | `listener_route_configs` | Listener→RouteConfig junction with `route_order` (RDS config naming). |
| Filter | `filters` | Reusable HTTP filter resource. `filter_type` is free TEXT (CHECK constraint **removed** in `migrations/20251210000001`; validated at app level by `FilterSchemaRegistry`). `source` `native_api\|ui\|openapi_import` (CHECK). Name unique per team. |
| Filter attachments | `route_config_filters`, `virtual_host_filters`, `route_filters`, `listener_filters` | Junctions with `filter_order` (unique per parent) and per-scope `settings` JSON (all except `listener_filters`). Inheritance: route_config < virtual_host < route (most specific wins). FK to `filters` is ON DELETE **RESTRICT** — a filter cannot be deleted while attached. |
| ListenerAutoFilter | `listener_auto_filters` | Bookkeeping rows recording HTTP filters auto-added to a listener because a filter was attached somewhere in its route tree. `attachment_level` `route_config\|virtual_host\|route` with CHECK enforcing which source-id columns are set. Used to remove listener HTTP filters when no source remains. |
| CustomWasmFilter | `custom_wasm_filters` | Team-uploaded WASM binary (BYTEA, ≤10 MB, magic `\0asm` = `[0x00,0x61,0x73,0x6d]`, SHA-256 stored) + JSON config schema. `runtime` default `envoy.wasm.runtime.v8`; `failure_policy` default `FAIL_CLOSED`. |
| Secret | `secrets` | SDS secret. Either encrypted-at-rest (`configuration_encrypted` AES-256-GCM + `nonce` + `encryption_key_id`, default key id `'default'`) or a reference to an external backend (`backend` ∈ vault/aws_secrets_manager/gcp_secret_manager/database — **no CHECK**, `reference`, `reference_version`; NULL backend = legacy DB-stored). `secret_type` CHECK `generic_secret\|tls_certificate\|certificate_validation_context\|session_ticket_keys`. Optional `expires_at`. |
| Dataplane | `dataplanes` | Envoy instance registration. `gateway_host` for MCP tool execution; cert tracking (`certificate_serial`, `certificate_expires_at` TEXT); `last_config_verify TIMESTAMPTZ` (agent liveness); `is_default` with partial unique index — **exactly one default dataplane per non-platform team** (invariant D8, `migrations/20260607000001`, backfilled with `gen_random_uuid()`). |
| ProxyCertificate | `proxy_certificates` | mTLS cert issuance audit (private keys never stored). `spiffe_uri` globally UNIQUE (`migrations/20260519000001`) because xDS binds mTLS connections via `get_by_spiffe_uri`. Revocation via `revoked_at/revoked_reason` (soft state, row kept). |
| XdsNackEvent | `xds_nack_events` | Persisted Envoy NACKs and warming-failure reports. `source` `stream\|warming_report` (CHECK); `nonce`/`version_rejected` nullable for warming reports; `dedup_hash` partial-unique for report dedup. |

### 2.6 Rate limiting (global RLS), audit, misc

| Entity | Table | Meaning |
|---|---|---|
| RateLimitDomain | `rate_limit_domains` | (org, team) ownership of an Envoy RLS `domain` string. **Soft delete** (`deleted_at`, reaped at 30d). | RateLimitPolicy | `rate_limit_policies` | (org, team, domain, descriptors) → `(requests_per_unit > 0, unit ∈ second\|minute\|hour\|day)`. `descriptors` JSONB flat string map (CHECK + trigger); trigger computes `descriptors_canonical` (sorted-key text via `jsonb_canonical_text`) used in the active-row unique index. | InstanceApp | `instance_apps` | Instance-wide feature flags keyed by `app_id` (e.g. `stats_dashboard`); `enabled INTEGER` 0/1 (not BOOLEAN), JSON `config`, `enabled_by/enabled_at` audit fields. |

### 2.7 Domain value types (not directly tables)

From `src/domain/` (full detail in agent-verified catalog; key invariants):

- **ClusterSpec.validate()**: name non-empty, ≥1 endpoint, `connect_timeout_seconds != 0`.
  `CircuitBreaker::default_config()` = 1024/1024/1024 conns/pending/requests, 3 retries.
  `OutlierDetection::default_config()` = consecutive_5xx 5, interval 10s, base ejection 30s,
  max ejection 10%, min hosts 1. `HealthCheck::http(path)` defaults interval 10s, timeout 5s,
  unhealthy 3, healthy 2. `LoadBalancingPolicy` ∈ RoundRobin, LeastRequest{choice_count},
  Random, RingHash{minimum_ring_size}, Maglev{table_size}.
- **ListenerSpec.validate()**: HTTPS ⇒ tls_config present; port ≠ 0; name non-empty.
  `Protocol` ∈ Http/Http2/Https/Tcp; `default_port()` 80/80/443/8080; `requires_tls()` = Https.
  `TlsVersion` ∈ V1_0..V1_3 (listener) vs V1_2/V1_3 (upstream).
- **SecretSpec.validate()** (`src/domain/secret.rs`): generic secret = non-empty valid base64;
  TLS cert/key non-empty and contain `-----BEGIN`; validation context trusted_ca PEM;
  session ticket keys decode to exactly **80 bytes**. SAN matchers tagged
  `match_type` ∈ exact/prefix/suffix/safe_regex/contains.
- **FilterType** (15 builtin, `src/domain/filter.rs`): header_mutation, jwt_auth, cors,
  compressor, local_rate_limit, rate_limit, ext_authz, rbac, oauth2, custom_response, mcp,
  health_check, ext_proc, rate_limit_quota, wasm. Static metadata per type:
  Envoy `http_filter_name`, `type_url`, `per_route_type_url`, attachment points
  (oauth2 & health_check are **listener-only**; everything else route+listener),
  `requires_listener_config` (true for all except header_mutation and cors),
  `per_route_behavior` ∈ `full_config` / `reference_only` (jwt_auth) /
  `disable_only` (compressor, mcp) / `not_supported` (oauth2, health_check).
  Custom (non-builtin) filter types may attach to route|listener, never cluster.
- **FilterSchemaRegistry** (`src/domain/filter_schema.rs`): 15 embedded YAML schemas
  (`include_str!`) + custom schemas loadable from directory or registered from
  `custom_wasm_filters`; hot-reloadable (`Arc<RwLock<…>>`); built-ins cannot be unregistered;
  schema validation: name/http_filter_name/type_url non-empty, attachment_points non-empty,
  config_schema must be a JSON object.
- **CustomWasmFilterSpec.validate()**: `name` matches `^[a-zA-Z0-9_-]+$`, display_name
  non-empty, config_schema is object, attachment_points non-empty; binary non-empty,
  ≤ `10*1024*1024` bytes, magic `[0x00,0x61,0x73,0x6d]`; sha256 hex stored.
- **Stats** (`src/domain/stats.rs`): `EnvoyHealthStatus::from_host_counts` — 0 total→unknown,
  100%→healthy, ≥50%→degraded, else unhealthy; any open circuit breaker ⇒ unhealthy.
- **EndpointHealthStatus.is_available()** = healthy|degraded.

## 3. Effective DB schema (after all 100 migrations)

Type/constraint notation is PostgreSQL. "team" semantics: since
`migrations/20260207000002` every `team` column stores **teams.id (UUID)**, not the name
(column was renamed back to `team` after conversion — beware: name says "team", content is id).
`proxy_certificates.team_id` and `grants.team_id` were id-based from the start;
`xds_nack_events.team` converted in `migrations/20260415000001`.

### 3.1 Org / user / auth

```sql
organizations (
  id TEXT PK, name TEXT NOT NULL UNIQUE, display_name TEXT NOT NULL, description TEXT,
  owner_user_id TEXT FK->users(id) ON DELETE SET NULL, settings TEXT,
  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','suspended','archived')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now, updated_at TIMESTAMPTZ NOT NULL DEFAULT now)
-- idx: name, status

organization_memberships (
  id TEXT PK, user_id TEXT NOT NULL FK->users CASCADE, org_id TEXT NOT NULL FK->organizations CASCADE,
  role TEXT NOT NULL DEFAULT 'member' CHECK (role IN ('owner','admin','member','viewer')),
  created_at TIMESTAMPTZ NOT NULL)
-- UNIQUE INDEX (user_id, org_id); idx: org_id

users (
  id TEXT PK, email TEXT NOT NULL UNIQUE, password_hash TEXT NOT NULL, name TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'active',                  -- no CHECK
  is_admin BOOLEAN NOT NULL DEFAULT FALSE,                -- platform superadmin
  zitadel_sub TEXT UNIQUE,
  user_type TEXT NOT NULL DEFAULT 'human' CHECK (user_type IN ('human','machine')),
  agent_context TEXT NULL CHECK (agent_context IN ('cp-tool','gateway-tool','api-consumer')),
  created_at, updated_at TIMESTAMPTZ NOT NULL)
-- org_id added 20260206000003 then DROPPED 20260302000002 (membership via organization_memberships)
-- idx: email, status, is_admin, (status,is_admin) WHERE is_admin, zitadel_sub, user_type,
--      agent_context WHERE NOT NULL

teams (
  id TEXT PK, name TEXT NOT NULL, display_name TEXT NOT NULL, description TEXT,
  owner_user_id TEXT FK->users SET NULL, settings TEXT,
  status TEXT NOT NULL DEFAULT 'active',                  -- no CHECK
  envoy_admin_port BIGINT,                                -- UNIQUE INDEX (global)
  org_id TEXT NOT NULL FK->organizations(id) ON DELETE RESTRICT,
  created_at, updated_at TIMESTAMPTZ NOT NULL,
  UNIQUE (org_id, name))                                  -- was globally UNIQUE(name) until 20260208000001
-- idx: status, owner_user_id, org_id, (org_id,name), (status,org_id,name), envoy_admin_port UNIQUE

user_team_memberships (
  id TEXT PK,
  user_id TEXT NOT NULL FK->users CASCADE,    -- NB: FK declared twice (inline + fk_user_team_memberships_user_id)

-- UNIQUE INDEX (user_id, team); idx: user_id, team

grants (                                                  -- 20260306000001
  id TEXT PK,
```

## 4. Invariants

### 4.1 Uniqueness

| Rule | Where |
|---|---|
| Cluster / RouteConfig / Listener names are **globally unique** (NOT per team) | `clusters.name UNIQUE`, `route_configs.name UNIQUE`, `listeners.name UNIQUE` — cross-team name collisions are possible attack/DoS surface; see §6 |
| `(name, version)` unique on clusters/route_configs/listeners | original 2024 migrations (vestigial: only one row per name ever exists; version bumps in place) |
| Listener `(address, port)` unique where port NOT NULL | global port collision guard across ALL teams |
| Team name unique per org; org name globally unique | `teams UNIQUE(org_id,name)`, `organizations.name UNIQUE` |
| `envoy_admin_port` globally unique across teams | `idx_teams_envoy_admin_port` |
| Per-team name uniqueness: filters, secrets, custom_wasm_filters, mcp_tools, dataplanes, import_metadata (spec_name), learning_sessions (name, partial) | `UNIQUE(team, name)` etc. |
| One default dataplane per team (partial unique), and exactly one required per non-platform team (D8) | `migrations/20260607000001` |
| `users.email` UNIQUE, `users.zitadel_sub` UNIQUE | |
| One membership per (user, team) and per (user, org) | unique indexes |
| Grants deduped via **partial** unique indexes (NULLs distinct in plain UNIQUE) | `migrations/20260304000002` comment explains rationale |
| `proxy_certificates.spiffe_uri` UNIQUE (mTLS identity binding must be a point lookup) | `migrations/20260519000001` |
| Filter order unique per attachment parent | `UNIQUE(parent_id, filter_order)` on all 4 junctions |
| `route_metadata` 1:1 with route | `UNIQUE(route_id)` |
| Schema versions: `UNIQUE(team, path, http_method, version)` + `previous_version_id` chain | aggregated_api_schemas |
| RLS: domain unique per org among active rows; policy unique per (org,team,domain,canonical descriptors) among active rows | partial indexes WHERE deleted_at IS NULL |

### 4.2 Referential / cascade rules (delete behavior)

- **Team delete:** RESTRICTed by `clusters`, `route_configs`, `listeners`, `filters`,
  `secrets`, `custom_wasm_filters` (core resources must be removed first); CASCADEs
  `learning_sessions`(→`inferred_schemas` via session), `aggregated_api_schemas`,
  `import_metadata`, `mcp_tools`, `dataplanes`, `user_team_memberships`,
  `proxy_certificates`, `xds_nack_events`, `grants`, rate-limit tables.
- **Org delete:** RESTRICTed by `teams.org_id`; CASCADEs `organization_memberships`,
  `grants`, rate-limit tables.
- **User delete:** CASCADEs memberships and `grants.principal_id`; SET NULL on
  `teams.owner_user_id`, `organizations.owner_user_id`, `proxy_certificates.issued_by_user_id`;
  **blocked** by `grants.created_by` (FK with no ON DELETE action).
- **Cluster delete:** CASCADEs `route_configs` via `cluster_name` FK (deleting a cluster
  silently deletes route configs that name it!), `cluster_endpoints`, `cluster_references`.
- **RouteConfig delete:** CASCADEs `virtual_hosts` → `routes` → `route_filters`,
  `route_config_filters`, `listener_route_configs`, `listener_auto_filters`,
  and via `routes`: `route_metadata`, `mcp_tools.route_id`, `grants.route_id`.
- **Filter delete:** RESTRICTed by all 4 attachment junctions (must detach first);
  `listener_auto_filters.source_filter_id` CASCADEs.
- **Import delete:** CASCADEs `route_configs.import_id` and `listeners.import_id`
  (resources die with the import) but `clusters.import_id` SET NULL (shared clusters survive;
  refcounted via `cluster_references.route_count`).
- **Dataplane delete:** `listeners.dataplane_id` SET NULL.

### 4.3 Versioning & delete style

- Gateway resources (`clusters`, `route_configs`, `listeners`, `filters`, `secrets`,
  `custom_wasm_filters`) carry `version BIGINT DEFAULT 1`, incremented by the repository on
  every UPDATE (`version = version + 1` in SQL). This drives xDS change detection. There is
  **no version-history table** — `UNIQUE(name, version)` is vestigial; old versions are
  overwritten in place. `audit_log.old_configuration/new_configuration` is the only history.
- **Soft delete exists ONLY in rate-limit tables** (`deleted_at` tombstones, 30-day reap,
  partial unique indexes scoped to active rows, optimistic-concurrency `version`). Everything
  else is hard DELETE. `proxy_certificates` uses revocation fields instead of deletion.
- `aggregated_api_schemas` versions by **row insertion** (immutable versions chained by
  `previous_version_id`), unlike gateway resources which version in place.
