## Codebase Patterns
- `docs/quickstart.md` was already deleted before this PRD started ‚Äî always verify file existence before planning deletion work
- The `expose` command source is on `release/v0.2.0` branch at `src/cli/expose.rs`, not on `main` ‚Äî Docker stack must be built from release/v0.2.0 for the expose API endpoint to work
- The installed `flowplane` binary at `~/.local/bin/flowplane` may have more commands than the current branch source ‚Äî always check `flowplane --help` against the live binary
- To test docs against a live stack when current branch lacks server features: `git stash && git checkout release/v0.2.0 && flowplane init ... && git checkout <branch> && git stash pop`
- Filter JSON uses camelCase for top-level API fields (`filterType`, not `filter_type`) ‚Äî the Rust struct has `#[serde(rename_all = "camelCase")]`
- Filter config is nested: `"config": {"type": "local_rate_limit", "config": {...actual settings...}}` ‚Äî custom Serialize/Deserialize on FilterConfig enum
- `local_rate_limit` is the implemented in-memory rate limiter; `rate_limit` (external/distributed) exists as FilterType but has `is_implemented: false`
- Listener naming convention from `expose`: `<name>-listener` ‚Äî needed for `filter attach --listener`
- MCP endpoints: `/api/v1/mcp/cp` (control plane) and `/api/v1/mcp/api` (gateway API) ‚Äî both Streamable HTTP
- `flowplane down` preserves database volume; `flowplane down --volumes` deletes it
- Prod mode uses `make up ENVOY=1 HTTPBIN=1` + `make seed` + `flowplane auth login` ‚Äî requires Zitadel
- Prod mode xDS port is 50051 (vs 18000 in dev mode)
- `flowplane list` strips the `-listener` suffix for display ‚Äî `demo-listener` shows as `demo`
- `flowplane status <name>` requires the full listener name (e.g., `demo-listener`), not the short display name
- `flowplane doctor` checks CP health and Envoy readiness at localhost:9901 ‚Äî skips Envoy check for remote servers
- `flowplane database` subcommands require `--database-url` or `DATABASE_URL` env var ‚Äî `init` handles migrations automatically
- Listener create requires `dataplaneId` ‚Äî dev mode uses `dev-dataplane-id`; expose handles this automatically
- Filter delete returns 409 if filter is still attached ‚Äî must detach first
- Cluster update JSON must include the `name` field (400 error otherwise)
- Listener filter chain type is `httpConnectionManager` (camelCase), NOT `HttpConnectionManager` (PascalCase)
- Rate limit denominator values are snake_case: `hundred`, `ten_thousand`, `million` ‚Äî NOT SCREAMING_CASE
- Dev mode token is stored at `~/.flowplane/credentials` (plain text), not in config.toml JWT
- Dev mode token env var: `FLOWPLANE_DEV_TOKEN` on the control plane container
